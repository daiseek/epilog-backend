# S3에 테스트용 PDF 업로드 방법

## 🎯 목적

- 로컬/배포 환경에서 동일한 PDF 파일로 부하테스트 수행
- Git에 PDF 파일을 올리지 않고도 일관된 테스트 환경 구축

## 📤 S3 업로드 단계

### 1. AWS S3 콘솔 접속

- https://s3.console.aws.amazon.com/ 접속
- 기존 프로젝트 S3 버킷 선택

### 2. 폴더 생성 (선택사항)

- `loadtest/` 폴더 생성하여 테스트 파일 분류

### 3. PDF 파일 업로드

- 테스트하고 싶은 PDF 파일들 업로드
- 권한: 퍼블릭 읽기 또는 프로젝트 IAM 권한에 맞게 설정

### 4. 객체 URL 복사

각 업로드된 파일의 URL 복사:

```
https://your-bucket.s3.region.amazonaws.com/loadtest/book1.pdf
https://your-bucket.s3.region.amazonaws.com/loadtest/book2.pdf
```

## ⚙️ 환경변수 설정

### 개발환경

`.env` 파일에 추가:

```
LOADTEST_PDF_URLS="https://your-bucket.s3.region.amazonaws.com/loadtest/book1.pdf,https://your-bucket.s3.region.amazonaws.com/loadtest/book2.pdf"
```

### 배포환경

`.env.prod` 파일에 추가:

```
LOADTEST_PDF_URLS="https://your-bucket.s3.region.amazonaws.com/loadtest/book1.pdf,https://your-bucket.s3.region.amazonaws.com/loadtest/book2.pdf"
```

### Docker Compose

`docker-compose.prod.yml`에 추가:

```yaml
environment:
  LOADTEST_PDF_URLS: "https://your-bucket.s3.region.amazonaws.com/loadtest/book1.pdf,https://your-bucket.s3.region.amazonaws.com/loadtest/book2.pdf"
```

## ✅ 확인 방법

부하테스트 실행 시 로그에서 확인:

```
🔧 환경변수에서 S3 PDF URL 로드: 2개
원격 URL: 2개
  1. your-bucket.s3.region.amazonaws.com
  2. your-bucket.s3.region.amazonaws.com
```

이제 실제 책 콘텐츠로 캐릭터 생성과 대본 생성을 테스트할 수 있습니다! 🎉
